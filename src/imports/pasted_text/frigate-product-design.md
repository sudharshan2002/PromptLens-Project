Create a premium product website and product UI system for a platform called FRIGATE.

Important:
Use the exact design language, motion philosophy, layout rhythm, and premium visual atmosphere of the live Kyma site as the reference style.
Do not make this generic SaaS.
Do not simplify it into a normal dashboard template.
This should feel like Kyma’s visual world transformed into a next-generation explainable AI product.

Brand / product concept:
Frigate is an evolution of PromptLens.
It solves the black-box problem in generative AI.
Users give prompts, but most AI tools do not show why outputs happen or how to improve them.
Frigate is a production-ready prompt-to-output engine with explainability built directly into the UX.

Core promise:
Users do not just use AI.
They understand, control, and trust it.

Product positioning:
Frigate should feel like:
- a creative cockpit
- a prompt debugger
- a co-creator that explains itself
- “Figma + ChatGPT + Explainable AI”

Design style:
Keep Kyma’s design DNA:
- acid-lime / black / cream palette
- editorial layout rhythm
- line-grid structure
- premium Framer-like motion
- cinematic section pacing
- sharp minimal card styling
- uppercase display typography
- mono utility text
- subtle but high-quality micro interactions

Color system:
- acid lime background: #D1FF00
- near-black text: #050505
- cream surfaces: #F4F4E8
- dark card surfaces: #1C1E19
- muted gray text: #686868
- off-white accents: #FFFFED
- thin translucent borders: #9C9C9C26

Typography:
- Major headings: TASA Orbiter style, uppercase, weight 800, tight tracking, strong editorial feel
- Body/UI: Geist or Inter
- Utility labels / tabs / metadata / panel labels: Roboto Mono
- Strong contrast between giant headings and small precision UI text

Create these main pages / experiences:
1. Home
2. Composer
3. What-If Analysis
4. Dashboard

The Composer is the main feature and must feel like the center of the product.

Core product features to communicate:

2. Real-Time What-If Editing

3. Explainability Layer
- Built into the generation pipeline, not optional
- Works for both text and image generation
- Text: token-level reasoning / influence
- Images: region-level influence / overlays

4. Trust + Insight Dashboard
- Tracks confidence, clarity, output quality, user satisfaction
- Provides feedback like:
  - “This word caused a style shift”
  - “This phrase added complexity”

Motion language:
Apply premium Framer-like motion throughout.
Use:
- large entrances: cubic-bezier(0.16, 1, 0.3, 1)
- small interactions: ease-out
- no bouncy motion
- no generic animation presets

Text animations:
- large headings reveal word-by-word or letter-by-letter
- start with slight blur and low opacity
- resolve to crisp full opacity
- small utility text should fade/slide upward subtly
- section labels, menu text, chips, metadata, footer labels should all have refined micro-animations

Use motion behavior like:
- color 0.3s linear
- filter 0.6s ease-out
- opacity 0.6s ease-out

Micro interactions:
- buttons: subtle lift, crisp hover, fast state change
- cards: slight translate and contrast shift
- tabs / chips / filters: elegant active states
- outputs and explanation overlays should animate progressively
- diff views should feel smooth and readable
- mapping overlays should fade in cleanly
- dashboard widgets should reveal with staggered motion
- footer elements should animate in with the same refined choreography
